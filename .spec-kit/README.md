# Spec-kit 사양 문서

이 디렉토리는 dev-agent-kit의 Spec-kit 기능을 사용하여 프로젝트의 사양 문서를 관리합니다.

## 구조

```
.spec-kit/
├── README.md              # 이 파일
├── requirements/          # 요구사항 문서
├── api/                   # API 스펙
├── architecture/          # 아키텍처 문서
├── features/              # 기능 스펙
└── changelog/             # 변경 이력
```

## 사용 방법

### 새 사양 문서 생성
1. 적절한 카테고리 디렉토리에 마크다운 파일 생성
2. 표준 템플릿 사용 (아래 참고)
3. 버전 정보 및 변경 이력 기록

### 사양 문서 템플릿

```markdown
# [문서 제목]

## 개요
[문서의 목적과 범위]

## 요구사항
[상세 요구사항]

## 설계
[설계 내용]

## 구현
[구현 세부사항]

## 테스트
[테스트 계획]

## 변경 이력
- YYYY-MM-DD: 초기 작성
```

## 버전 관리

모든 사양 문서는 Git으로 버전 관리됩니다.
주요 변경사항은 changelog/ 디렉토리에 기록하세요.
