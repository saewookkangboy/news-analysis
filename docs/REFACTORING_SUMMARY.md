# 프로젝트 구조 개편 요약

## 📋 개편 목적
서비스 유지보수 및 운영 편의성을 위해 파일 구조와 데이터 연결 구조를 개선했습니다.

## 🔄 주요 변경 사항

### 1. 문서 파일 정리
- **변경 전**: 루트 디렉토리에 20개 이상의 문서 파일이 산재
- **변경 후**: 모든 문서를 `docs/` 폴더로 통합
- **효과**: 프로젝트 루트가 깔끔해지고 문서 관리가 용이해짐

### 2. 환경 변수 관리 개선
- **추가**: `.env.example` 파일 생성
- **효과**: 
  - 새로운 개발자가 쉽게 환경 변수 설정 가능
  - 필요한 환경 변수 목록을 한눈에 확인 가능
  - Git에 실제 키가 포함되지 않도록 보호

### 3. 데이터 디렉토리 구조 명확화
- **개선 사항**:
  - `backend/config.py`에서 디렉토리 경로 명확히 정의
  - `exports/` 디렉토리 추가 (분석 결과 내보내기용)
  - `ASSETS_DIR` 경로를 `frontend/public/assets`로 명확화
- **효과**: 데이터 저장 위치가 명확해지고 관리가 용이해짐

### 4. Vercel 설정 개선
- **변경**: `vercel.json`에 명확한 라우팅 규칙 추가
- **효과**: 배포 시 라우팅이 예측 가능하고 안정적

### 5. 프로젝트 구조 문서화
- **추가**: `docs/PROJECT_STRUCTURE.md` 생성
- **내용**:
  - 상세한 디렉토리 구조 설명
  - 데이터 연결 구조 설명
  - 환경별 차이점 설명
  - 유지보수 가이드

### 6. .gitignore 개선
- **개선**: 더 포괄적인 무시 규칙 추가
- **효과**: 불필요한 파일이 Git에 포함되지 않음

## 📁 새로운 디렉토리 구조

```
news-trend-analyzer/
├── docs/              # 모든 문서 파일 (새로 생성)
├── data/              # 데이터 저장소
│   ├── raw/          # 원시 데이터
│   ├── processed/    # 처리된 데이터
│   └── cache/        # 캐시 데이터
├── exports/           # 분석 결과 내보내기 (새로 추가)
└── logs/             # 로그 파일
```

## 🔗 데이터 연결 구조 개선

### 설정 관리 중앙화
- 모든 설정은 `backend/config.py`에서 관리
- 환경 변수는 `.env` 또는 시스템 환경 변수에서 로드
- 템플릿 파일 `.env.example` 제공

### API 라우팅 명확화
- 로컬 개발: `backend/main.py` 직접 실행
- Vercel 배포: `api/index.py`를 Serverless Function으로 사용
- 루트 `index.py`: 선택적 진입점 (주석 명확화)

### 서비스 계층 구조
- 각 서비스는 독립적으로 관리
- 명확한 책임 분리
- 재사용 가능한 구조

## 🚀 사용 방법

### 새로운 개발자 온보딩
1. 저장소 클론
2. `.env.example`을 `.env`로 복사
3. `.env`에 실제 API 키 입력
4. `pip install -r requirements.txt`
5. `python run.py` 실행

### 환경 변수 설정
```bash
# 템플릿 복사
cp .env.example .env

# 편집기로 열어서 실제 값 입력
nano .env  # 또는 원하는 편집기
```
